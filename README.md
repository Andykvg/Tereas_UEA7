## *Objetivo* 
El objetivo de este repositorio es el de almacenar los distintos ejercicios y tareas que se realicen durante las clases del eje tecnológico de la UEA 7 "Producción y comunicación colaborativa". Se pretende organizar los trabajos con el fin no solo de utilizarlos durante el transcurso de este trimestre, sino también para utilizar el repositorio como recurso de apoyo en futuros proyectos. Además, y como objetivo puntual, el repositorio contendrá la descripción de los proyectos, la cual agrega un nivel de organización que, al momento de ser recuperados los códigos, se podrá consultar con detenimiento cada una de sus funciones. 

## *Organización*
La organización de este repositorio se maneja de la siguiente manera:

* Carpeta

La carpeta se nombra según el ejercicio o tarea que se asigna durante el eje tecnológico. Después, dentro de la carpeta se encuentran los archivos que se necesitan para hacer funcionar el proyecto; hasta el momento no se ha agregado una descripción de cada carpeta, pero se tiene contemplado para futuros trabajos, así como comentar cada código. Podemos decir que mayormente las carpetas contienen un archivo: html, css y un javascript, aunque recientemente se han trabajado con archivos json. 

 * HTML: Es la base de todos los proyectos y se complementa con alguno de los demás archivos.

	* CSS: Se utiliza como apoyo para el diseño de la página web.

	* JavaScript: Genera interactividad con la web y ayuda agregando funciones dentro de la consola.

	* Json: Cuenta con distintas funciones; sin embargo, actualmente es un recurso que nos apoya con la conexión con Supabase. 


## *Experiencia*
Por último, en este espacio describiré cómo ha sido mi experiencia utilizando y gestionando un repositorio en GitHub. Uno de los primeros puntos que quisiera comentar es mi dificultad para comprender completamente la interfaz de GitHub, puesto que, a pesar de haberlo utilizado en trimestres anteriores, admito no haber explorado todas las funciones de la aplicación, además de haber perdido la práctica en ciertos trimestres. Sin embargo, considero a GitHub un espacio con una gran variedad de posibilidades y dinámicas, ideal para trabajar con proyectos colaborativos, como ha sido el tema principal del trimestre en curso. Gracias a la aplicación es posible compartir y recibir código en tiempo real, aunque cabe aclarar que algunas funciones son poco intuitivas; la práctica y la investigación logran resolver los problemas principales. 

En cuanto al envío de tareas, considero que la aplicación ha facilitado el envío de código, ya que anteriormente esto significaba un problema, debido al peso de los archivos. Y cabe resaltar las nuevas funciones que hemos aprendido en el curso, como lo son la gestión de tareas en un espacio de trabajo, la creación de una organización, así como la implementación de otras aplicaciones como lo son Supabase. 

Como conclusión, mis experiencias en GitHub me han parecido productivas, más en específico en este repositorio que me ha ayudado a organizar mi trabajo. Considero seguir utilizando este software no solo lo que queda en mi carrera académica, sino también como parte de mi vida laboral.
